[dataset] Write writer.py — Cohere API document synthesis #17

What this does
This is Step 2 of the dataset pipeline. facts.py (#28) has already decided what goes into each document — the rent amount, notice period, salary, permit type, etc. Your job is to take those facts and ask Aya Expanse 35B via the Cohere API to write a realistic legal document from them.
Think of it like this: facts.py hands you a blueprint that says "German lease, rent = 1350 EUR, no pets, 60 days notice." You hand that blueprint to Cohere and say "write me a realistic German rental agreement that contains all of these facts." The output is a full document string that reads like something a real tenant would receive.
This is the only step in the pipeline that needs the internet — Cohere API credits are confirmed available.
Scale: 4 domains × 3 languages × 10 documents = 120 documents total
How to call facts.py:

from dataset.builder.facts import generate_facts, SUPPORTED_DOMAINS
# SUPPORTED_DOMAINS = ["lease", "employment", "health_insurance", "immigration_letter"]

for domain in SUPPORTED_DOMAINS:
    for lang in ["de", "hi", "sw"]:
        for doc_idx in range(10):
            facts = generate_facts(lang, domain, doc_idx)
            # facts is a plain dict — pass it to Cohere to generate document text
            # facts["_language"]       → "de" / "hi" / "sw"
            # facts["_domain"]         → "lease" / "employment" etc.
            # facts["_document_index"] → 0–9
            # facts["_seed"]           → int, store in JSONL for auditability
Document length: Please add variability in document length — vary between 1 and 5 pages per document. Real legal documents vary significantly in length and this makes our pipeline more robust.
