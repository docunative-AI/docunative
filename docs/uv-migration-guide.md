# UV Build System Migration Guide

## Overview

DocuNative has migrated from conda-based dependency management to **UV**, a modern, fast Python package manager. This document explains the changes, benefits, and how to work with the new system.

---

## What Changed?

### Before: Conda Export (471 packages)

The old `requirements.txt` was generated by conda and contained:
- ❌ 471 packages (direct + transitive dependencies)
- ❌ Local filesystem paths (`@ file:///private/var/folders/...`)
- ❌ Machine-specific (could not be installed on other systems)
- ❌ Mixed build tools, IDEs, and project dependencies
- ❌ No version locking mechanism

### After: UV with pyproject.toml (9 packages)

The new `pyproject.toml` contains:
- ✅ 9 essential direct dependencies only
- ✅ Standard PyPI package names (no local paths)
- ✅ Python 3.11 pinned for stability
- ✅ `uv.lock` file for reproducible installs
- ✅ 98% smaller dependency declaration
- ✅ 10-100x faster installs

## Python 3.11: Why This Version?

The project is now pinned to Python 3.11 specifically:

```toml
requires-python = ">=3.11,<3.12"
```

### Rationale

| Factor | Benefit |
|--------|---------|
| **ML Library Stability** | Best compatibility with torch, transformers, sentence-transformers |
| **Performance** | ~25% faster than Python 3.10 (PEP 659 optimizations) |
| **Proven Track Record** | Stable release with mature ecosystem support |
| **Chromadb Compatibility** | Fully tested with chromadb's native dependencies |
| **Avoids Bleeding Edge** | Python 3.12+ has breaking changes in some ML deps |

### What About Python 3.12+?

⚠️ **Not recommended yet** because:
- Some ML dependencies have compatibility issues
- `sentence-transformers` and `chromadb` have known quirks on 3.12
- 3.11 is the current "stable LTS-like" choice for ML projects

## Installation Guide

### Prerequisites

1. **Install UV** (if not already installed):

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Or via Homebrew (macOS)
brew install uv
```

2. **Ensure Python 3.11 is available** on your system:

```bash
# macOS (via Homebrew)
brew install python@3.11

# Linux (via apt)
sudo apt install python3.11 python3.11-venv

# Windows
# Download from python.org or use pyenv-windows
```

### Fresh Install

```bash
# Clone the repo
git clone https://github.com/docunative-AI/docunative.git
cd docunative

# Install everything (creates .venv with Python 3.11)
make install
```

**What happens:**
1. UV creates a Python 3.11 virtual environment in `.venv/`
2. UV reads `pyproject.toml` and resolves all dependencies
3. UV generates `uv.lock` (pinned versions for reproducibility)
4. UV installs all packages (~2-5 minutes on first run)

### Migrating from Old Setup

If you have an existing conda environment:

```bash
# Deactivate conda
conda deactivate

# Remove old .venv if it exists
rm -rf .venv

# Run the new install
make install

# Activate the new environment
source .venv/bin/activate
```

## Using UV in Development

### Running Commands

```bash
# Install dependencies
make install

# Run tests
make test

# Launch the UI
make demo

# Or run any Python script directly with uv run
uv run python pipeline/extract.py

# Run with activated venv (traditional way)
source .venv/bin/activate
python ui/app.py
```

### Adding New Dependencies

If you need to add a new package:

1. **Edit `pyproject.toml`**:

```toml
dependencies = [
    # ... existing deps ...
    "new-package>=1.0,<2.0",
]
```

2. **Sync the environment**:

```bash
uv sync
```

UV will automatically update `uv.lock` with the new dependency tree.

### Updating Dependencies

```bash
# Update all packages to latest compatible versions
uv lock --upgrade

# Then sync
uv sync
```

## The uv.lock File

The `uv.lock` file is **auto-generated** and should be **committed to git**.

### What It Does

- Pins exact versions of all packages (direct + transitive)
- Ensures every developer gets identical dependencies
- Like `package-lock.json` (npm) or `Cargo.lock` (Rust)

### Example

```toml
[[package]]
name = "torch"
version = "2.3.1"
source = { registry = "https://pypi.org/simple" }
dependencies = [
    { name = "filelock" },
    { name = "typing-extensions" },
    { name = "sympy" },
    { name = "networkx" },
    { name = "jinja2" },
    { name = "fsspec" },
]
```

### Git Workflow

```bash
# DO commit uv.lock
git add uv.lock pyproject.toml
git commit -m "Update dependencies"

# DON'T commit .venv/
# (already in .gitignore)
```

## Troubleshooting

### Metal Acceleration (macOS)

If torch doesn't detect Apple Silicon GPU:

```bash
# Check MPS availability
uv run python -c "import torch; print(torch.backends.mps.is_available())"
```

If `False`, reinstall torch with Metal support:

```bash
uv pip install --upgrade --force-reinstall torch
```

### CUDA Support (Linux/Windows)

For NVIDIA GPU acceleration:

```bash
# Check CUDA availability
uv run python -c "import torch; print(torch.cuda.is_available())"
```

If `False`, ensure CUDA drivers are installed. UV installs the CPU-only torch by default.

For CUDA support, you may need to use a different torch index:

```bash
uv pip install torch --index-url https://download.pytorch.org/whl/cu118
```

### "Python 3.11 not found"

If UV can't find Python 3.11:

```bash
# Specify the path explicitly
uv venv --python /usr/bin/python3.11

# Or use pyenv
pyenv install 3.11.8
pyenv local 3.11.8
uv venv
```

### Permission Errors

On Linux/macOS, if you get permission errors:

```bash
# Don't use sudo with uv
# Instead, ensure your user owns the project directory
chown -R $USER:$USER .
```

### Install Speed

| Tool | Time (Fresh Install) | Time (Cached) |
|------|---------------------|---------------|
| pip | ~8-12 minutes | ~4-6 minutes |

| Tool | Resolver | Speed |
|------|----------|-------|

## FAQ

### Can I still use conda?

Yes, but it's not recommended. The `requirements.conda-export.txt.bak` file is preserved for reference, but it won't be maintained going forward.

### What about virtual environments?

UV manages its own `.venv/` directory automatically. You don't need to run `python -m venv` manually.

### Do I need to activate the venv?

Not if you use `uv run` or `make` commands. But you can still activate it traditionally:

```bash
source .venv/bin/activate  # macOS/Linux
.venv\Scripts\activate     # Windows
```

### How do I uninstall everything?

```bash
# Remove the virtual environment
rm -rf .venv

# Remove UV cache (optional, saves ~500MB)
rm -rf ~/.cache/uv
```

### Where is uv.lock stored?

In the project root, next to `pyproject.toml`. Always commit it to git.
